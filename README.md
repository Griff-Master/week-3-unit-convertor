:rocket: UNIT CONVERTOR

:rocket: How the Program works
  
1. Allows user input
2. Allows user to do conversions of
    - Length 
    - Temperature
    - Currency
    - Mass
    -user is required to choose their respective convertor in main menu
3. Each convertor has several option the user can choose from
4. Each convertor allows going back to main menu
5. The program handle well invalid inputs and does not crash 
6. The results are formated to two decimal places
7. The program allows display of conversion history for each convertor
8. The program rejects entry of negative values in conversion
9. The program is menu-driven and keeps on running until user chooses to exit

:hammer_and_wrench: Methods used
- main method
- showMainMenu()
- lengthConvertor()
- temperatureConvertor()
- currencyConvertor()
- massConvertor()

:brain: Reflection Questions
1. Why is double most suitable than int in this program
    - Answer ➡️ double will store decimal numbers and give accurate result
2. Why is it important to place conversion logic inside methods?
    - Answer ➡️ For organization as this makes the program easy to read and for reusability as you can call a method
3. What advantage does a loop provide in a menu-driven application
    - the loop allows the program to continue running util user chooses to exit
4. Why is input validation important in user-facing programs?
    - Answer➡️ prevents invalid values and program crashes
5. How could this application be improved later using object-oriented programming?
    - Answer ➡️ Create converter classes instead of everything in main
